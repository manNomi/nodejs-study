# 익스프레스 웹 서버 만들기 강의 03. Router 모듈화

## 학습 개요
Router 모듈화를 통해 Express 애플리케이션을 계층적으로 설계하는 기준을 정리한다.

## 핵심 개념
### 1) 미들웨어 중심 아키텍처
Express의 본질은 미들웨어 체인이다. 요청이 어떤 순서로 처리되는지 이해해야 인증, 검증, 에러 처리가 안정된다.

### 2) 라우터와 비즈니스 분리
컨트롤러/서비스/저장소 경계를 나누면 테스트 가능성과 유지보수성이 크게 올라간다.

### 3) 운영 관점 설계
Router 모듈화는 기능 구현뿐 아니라 에러 표준화, 보안 미들웨어, 입력 검증 전략까지 함께 고려해야 완성된다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트 요구사항을 API 계약으로 빠르게 반영할 수 있다.
- 에러 응답을 일관화해 UI 예외 처리 복잡도를 낮춘다.
- 인증/권한 흐름을 화면 라우팅과 정합성 있게 맞출 수 있다.

## 실무에서 자주 생기는 오해
- 라우터에 비즈니스 로직을 과도하게 넣는다.
- 중앙 에러 핸들러 없이 try/catch를 중복 작성한다.
- 검증/인증/권한 순서를 섞어 보안 취약점을 만든다.

## 정리
Router 모듈화는 Express 코드를 '동작하는 코드'에서 '운영 가능한 구조'로 전환하는 핵심이다.
