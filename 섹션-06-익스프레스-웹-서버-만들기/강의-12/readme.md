# 익스프레스 웹 서버 만들기 강의 12. 보안 미들웨어

## 학습 개요
보안 미들웨어는 Express 애플리케이션을 계층적으로 구성하는 기준을 설명한다. 요청 처리 흐름을 구조화해 유지보수성을 높이는 것이 목적이다.

## 왜 중요한가
기능이 늘어날수록 라우터, 서비스, 검증, 에러 처리가 섞인다. 초기 구조 설계가 없으면 확장 단계에서 급격히 복잡해진다.

## 핵심 개념
### 1) 미들웨어 중심 구조
Express의 본질은 미들웨어 체인이다. 처리 순서를 정확히 설계해야 인증, 권한, 검증, 로깅이 일관된다.

### 2) 계층 분리
라우터/컨트롤러/서비스/저장소 경계를 분리하면 테스트 범위와 책임이 명확해진다.

### 3) 운영 품질 관점
보안 미들웨어를 다룰 때는 기능 구현뿐 아니라 에러 표준화, 입력 검증, 보안 설정까지 함께 고려해야 완성된다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트 요구사항을 API 계약으로 빠르게 변환할 수 있다.
- UI에서 처리할 에러 케이스를 서버 응답 표준과 연결할 수 있다.
- 인증 상태 흐름을 화면 라우팅과 정합성 있게 설계할 수 있다.

## 실무에서 자주 생기는 오해
- 라우터에 비즈니스 로직을 집중시킨다.
- 중앙 에러 핸들러 없이 예외 처리를 분산한다.
- 검증/인증/권한 순서를 섞는다.

## 정리
보안 미들웨어는 Express 코드를 운영 가능한 서비스 구조로 전환하는 핵심 단원이다.
