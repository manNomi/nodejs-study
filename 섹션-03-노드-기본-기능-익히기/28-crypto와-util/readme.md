# 28. crypto와 util

## 학습 개요
crypto와 util는 Node 코어 기능을 중심으로, 서버가 운영체제 자원을 어떻게 다루는지 설명한다. 이 섹션을 통해 Node.js를 단순 API 서버 도구가 아니라 시스템 도구로 이해하게 된다.

## 왜 중요한가
실무 병목은 대부분 파일 I/O, 메모리, 프로세스, 모듈 구조에서 발생한다. 코어 모듈 이해가 없으면 원인 분석이 어렵다.

## 핵심 개념
### 1) 코어 모듈의 책임
fs/path/process/stream/crypto 등 내장 모듈은 서버의 기본 동작 단위를 담당한다. 외부 라이브러리 이전에 이 계층을 이해해야 설계 판단이 가능하다.

### 2) 성능과 자원 관리
스트림, 버퍼, 워커 스레드, child_process는 성능 최적화 수단이자 장애 예방 수단이다. 어떤 상황에 무엇을 써야 하는지가 핵심이다.

### 3) 모듈 경계와 구조화
CommonJS/ESM, 순환참조, 의존 경계는 유지보수 비용을 좌우한다. crypto와 util를 통해 코드 구조 품질까지 함께 관리해야 한다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트에서 보던 응답 지연을 서버 자원 관점으로 분석할 수 있다.
- 대용량 데이터 처리에서 메모리 전략을 설계할 수 있다.
- 운영 로그와 에러를 구조적으로 정리하는 기준이 생긴다.

## 실무에서 자주 생기는 오해
- 요청 처리 경로에서 동기 API를 남용한다.
- 스트림 대신 전체 메모리 적재 방식을 고정 사용한다.
- 모듈 경계 없이 기능을 붙여 기술부채를 키운다.

## 정리
crypto와 util는 Node.js 백엔드를 '동작'에서 '운영' 단계로 올리는 코어 단원이다.
