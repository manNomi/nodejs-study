# 22. exports this require 순환참조

## 학습 개요
exports this require 순환참조를 포함한 Node 코어 기능을 통해 서버가 OS 자원을 어떻게 다루는지 정리한다.

## 핵심 개념
### 1) Node 코어 모듈의 역할
fs, path, process, stream, crypto 같은 코어 모듈은 외부 라이브러리 없이도 서버 핵심 기능을 구성할 수 있게 해준다.

### 2) 성능과 자원 관리
파일 I/O, 버퍼/스트림, 워커 스레드, 자식 프로세스는 성능과 메모리 효율을 결정한다. 사용 기준을 모르면 병목이 쉽게 발생한다.

### 3) 모듈 시스템과 구조화
CommonJS/ESM 이해는 코드 분리, 재사용, 순환참조 회피 같은 설계 품질과 직결된다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트에서 보던 느린 API의 원인을 서버 자원 모델로 분석할 수 있다.
- 대용량 데이터 처리에서 스트림 중심 설계를 선택할 수 있다.
- 운영 로그와 에러 처리 기준을 코드 레벨에서 통일할 수 있다.

## 실무에서 자주 생기는 오해
- 동기 API를 요청 처리 경로에서 남용한다.
- 메모리 사용량 확인 없이 전체 데이터를 한 번에 읽는다.
- 모듈 경계 없이 코드가 커져 유지보수 비용이 급증한다.

## 정리
exports this require 순환참조는 Node.js를 단순 서버 프레임워크가 아니라 시스템 프로그래밍 도구로 이해하게 만드는 장이다.
