# 패키지 매니저 강의 01. npm 생태계와 package.json 구조

## 학습 개요
npm 생태계와 package.json 구조를 중심으로 Node 프로젝트의 의존성/실행/배포 관리 체계를 정리한다.

## 핵심 개념
### 1) 패키지 관리의 기본 단위
package.json은 단순 메타데이터 파일이 아니라 프로젝트 실행 계약서다. 스크립트와 의존성 규칙이 팀 생산성을 좌우한다.

### 2) 버전과 재현 가능성
semver와 lockfile을 이해해야 개발/테스트/배포 환경을 일치시킬 수 있다.

### 3) 자동화 관점
npm 생태계와 package.json 구조를 정리하면 로컬 실행, CI, 배포 파이프라인까지 같은 명령 체계를 공유할 수 있다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트/백 공통으로 npm 스크립트 계약을 맞출 수 있다.
- 환경 불일치로 생기는 버그를 줄일 수 있다.
- 온보딩 시간을 크게 단축할 수 있다.

## 실무에서 자주 생기는 오해
- 버전 범위를 무분별하게 넓혀 예측 불가능한 빌드를 만든다.
- 스크립트를 문서화하지 않아 팀 내 실행 방식이 분산된다.
- dev/prod 의존성을 구분하지 않는다.

## 정리
npm 생태계와 package.json 구조는 코드 이전에 개발 체계를 안정화하는 단원이다.
