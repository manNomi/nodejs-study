# 패키지 매니저 강의 04. 패키지 배포/재사용 전략

## 학습 개요
패키지 배포/재사용 전략는 Node 프로젝트의 실행·의존성·배포 관리 체계를 설명한다. 코드 품질 못지않게 환경 재현성이 중요하다는 점을 다룬다.

## 왜 중요한가
같은 코드가 환경마다 다르게 동작하면 디버깅 비용이 급격히 커진다. 패키지 관리 규칙은 팀 생산성의 핵심이다.

## 핵심 개념
### 1) package.json의 계약 역할
package.json은 단순 설정 파일이 아니라 프로젝트 실행 계약이다. 스크립트와 의존성 정의가 협업 기준이 된다.

### 2) 버전 전략과 lockfile
semver 범위와 lockfile 정책을 이해해야 로컬/CI/운영 환경을 일관되게 맞출 수 있다.

### 3) 자동화 연결
패키지 배포/재사용 전략를 정리하면 개발, 테스트, 배포를 같은 명령 체계로 통합할 수 있다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트/백 공통 스크립트 체계를 맞춰 협업 속도를 높일 수 있다.
- 의존성 충돌 이슈를 사전에 줄일 수 있다.
- 온보딩 시간을 단축할 수 있다.

## 실무에서 자주 생기는 오해
- 버전 범위를 무분별하게 넓힌다.
- 실행 스크립트를 문서화하지 않는다.
- 개발/운영 의존성을 구분하지 않는다.

## 정리
패키지 배포/재사용 전략는 코드 실행 환경을 안정적으로 통제하기 위한 관리 단원이다.
