# 18. 옵셔널 체이닝 널 병합

## 학습 개요
옵셔널 체이닝 널 병합를 중심으로 백엔드 코드에서 반드시 필요한 JavaScript 런타임 동작 원리를 정리한다.

## 핵심 개념
### 1) 실행 순서와 컨텍스트
호출 스택, 스코프, this, 클로저 같은 개념은 서버 로직의 안정성을 결정한다. 특히 핸들러 체인에서 컨텍스트 오해는 치명적인 버그로 이어진다.

### 2) 비동기 제어 흐름
Promise, async/await, 이벤트 루프를 정확히 이해해야 DB/네트워크 요청 처리에서 경쟁 상태를 줄일 수 있다.

### 3) 현대 문법의 실무 의미
구조분해, 클래스, optional chaining 등은 단순 문법이 아니라 코드 가독성과 오류 예방에 직접적인 영향을 준다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 에러 처리를 UI 레벨이 아닌 비동기 흐름 전체에서 설계할 수 있다.
- 요청 병렬/직렬 전략을 코드로 명확히 구분할 수 있다.
- 팀 코드에서 일관된 문법과 스타일 기준을 세울 수 있다.

## 실무에서 자주 생기는 오해
- Promise 체인을 섞어 쓰며 예외 전파 경로를 잃는다.
- var/let/const 차이를 무시해 스코프 버그를 만든다.
- 문법만 외우고 실행 순서를 검증하지 않는다.

## 정리
옵셔널 체이닝 널 병합는 Node.js 백엔드의 문법 장이 아니라, 런타임 동작을 제어하는 안전장치에 가깝다.
