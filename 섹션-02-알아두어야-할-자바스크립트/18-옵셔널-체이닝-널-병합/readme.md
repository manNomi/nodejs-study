# 18. 옵셔널 체이닝 널 병합

## 학습 개요
옵셔널 체이닝 널 병합는 백엔드 코드에서 반드시 필요한 자바스크립트 런타임 동작을 정리하는 장이다. 문법 암기가 아니라 실행 순서와 상태 전이를 이해하는 데 초점을 둔다.

## 왜 중요한가
API 서버는 비동기 흐름이 많기 때문에 호출 순서, 스코프, 에러 전파를 제대로 이해하지 않으면 간헐적 버그가 빠르게 늘어난다.

## 핵심 개념
### 1) 실행 순서와 컨텍스트
호출 스택, 실행 컨텍스트, 스코프는 요청 처리 흐름을 예측하는 기본 도구다. 서버에서는 이 예측 가능성이 안정성으로 직결된다.

### 2) 비동기 흐름 제어
Promise/async-await의 핵심은 가독성이 아니라 제어권 관리다. 언제 기다리고, 언제 병렬 처리할지 결정하는 기준이 필요하다.

### 3) 안전한 현대 문법 사용
구조분해, optional chaining, nullish coalescing 같은 문법은 런타임 오류를 줄이고 코드 계약을 명확하게 만든다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트에서 쓰던 비동기 문법을 서버 I/O 패턴에 맞게 재해석할 수 있다.
- 에러 처리 기준을 컴포넌트 단위가 아니라 요청 흐름 단위로 옮길 수 있다.
- 팀 내 코드 스타일을 안정성 중심으로 통일할 수 있다.

## 실무에서 자주 생기는 오해
- Promise와 async-await를 혼용하며 예외 전파를 깨뜨린다.
- 스코프 규칙을 무시해 핸들러 상태가 오염된다.
- 문법을 알지만 실행 순서를 검증하지 않는다.

## 정리
옵셔널 체이닝 널 병합는 Node.js 백엔드에서 버그를 줄이는 가장 현실적인 기반 지식이다.
