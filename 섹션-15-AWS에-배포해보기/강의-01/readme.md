# AWS에 배포해보기 강의 01. AWS 배포 아키텍처 이해

## 학습 개요
AWS 배포 아키텍처 이해를 통해 AWS 운영 배포 체계와 장애 대응 기준을 정리한다.

## 핵심 개념
### 1) 배포 파이프라인 전체 그림
빌드, 배포, 헬스체크, 모니터링, 롤백이 하나의 체인으로 설계되어야 운영 안정성이 확보된다.

### 2) 운영 관측 가능성
로그/메트릭/알람 체계가 없으면 장애 원인 분석 속도가 급격히 떨어진다.

### 3) 복구 가능성 중심 설계
AWS 배포 아키텍처 이해의 핵심은 '배포 성공'이 아니라 '문제 발생 시 빠르게 되돌릴 수 있는 구조'를 만드는 것이다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트 배포와 백엔드 배포 타이밍을 정합성 있게 맞출 수 있다.
- 장애 시 UI 이슈와 서버 이슈를 로그로 빠르게 분리할 수 있다.
- 환경변수/시크릿 관리 원칙을 팀 공통으로 정리할 수 있다.

## 실무에서 자주 생기는 오해
- 헬스체크 자동화를 생략한다.
- 모니터링 없이 수동 점검에 의존한다.
- 롤백 절차 문서 없이 배포를 반복한다.

## 정리
AWS 배포 아키텍처 이해는 코드 배포를 운영 품질 관리 체계로 확장하는 핵심 단원이다.
