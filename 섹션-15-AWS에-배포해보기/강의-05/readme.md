# AWS에 배포해보기 강의 05. 로그/모니터링

## 학습 개요
로그/모니터링는 AWS 환경에서 배포와 운영을 하나의 체계로 연결하는 방법을 설명한다.

## 왜 중요한가
배포 성공 자체는 목표가 아니다. 장애를 빠르게 감지하고 되돌릴 수 있어야 실제 운영 품질이 확보된다.

## 핵심 개념
### 1) 배포 파이프라인
빌드, 배포, 헬스체크, 트래픽 전환 단계를 분리해 설계해야 실패 지점을 빠르게 찾을 수 있다.

### 2) 관측 가능성
로그/메트릭/알람이 있어야 UI 이슈와 서버 이슈를 분리 분석할 수 있다.

### 3) 복구 전략
로그/모니터링에서는 롤백 절차와 운영 문서가 코드만큼 중요하다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트/백 배포 타이밍을 정합성 있게 조정할 수 있다.
- 운영 장애 시 화면 문제와 서버 문제를 빠르게 분리한다.
- 시크릿/환경변수 관리 원칙을 팀 규칙으로 만들 수 있다.

## 실무에서 자주 생기는 오해
- 헬스체크 자동화를 생략한다.
- 모니터링 없이 수동 점검에 의존한다.
- 롤백 문서 없이 배포를 진행한다.

## 정리
로그/모니터링는 코드 배포를 운영 체계로 확장하는 핵심 단원이다.
