# http 모듈로 서버 만들기 강의 04. 정적 파일 서빙과 캐싱

## 학습 개요
정적 파일 서빙과 캐싱를 통해 프레임워크 이전 단계의 HTTP 본질을 이해하고 서버 기본기를 다진다.

## 핵심 개념
### 1) HTTP 프로토콜의 구성
요청 라인, 헤더, 바디, 상태코드를 정확히 이해해야 API 설계의 일관성을 유지할 수 있다.

### 2) 요청 처리 파이프라인
요청 수신부터 응답 반환까지 흐름을 직접 다루면 프레임워크가 내부에서 해주는 일을 명확히 이해할 수 있다.

### 3) 표준 응답 규약
정적 파일 서빙과 캐싱를 학습할 때 가장 중요한 산출물은 코드 자체보다 응답 형식과 에러 규약의 표준화다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트에서 필요한 응답 구조를 서버에서 먼저 정의할 수 있다.
- 상태코드와 메시지 규칙을 맞춰 디버깅 시간을 줄일 수 있다.
- API 실패 케이스를 UI 설계 단계에서 예측할 수 있다.

## 실무에서 자주 생기는 오해
- 상태코드 의미를 무시하고 200 응답만 사용한다.
- 입력 검증 없이 요청 본문을 신뢰한다.
- 예외 경로를 설계하지 않아 장애 시 응답이 깨진다.

## 정리
정적 파일 서빙과 캐싱는 프레임워크 의존도를 낮추고 HTTP 본질을 이해하는 핵심 단원이다.
