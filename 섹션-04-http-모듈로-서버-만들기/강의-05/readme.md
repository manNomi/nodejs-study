# http 모듈로 서버 만들기 강의 05. POST 데이터 처리와 검증

## 학습 개요
POST 데이터 처리와 검증는 프레임워크 이전 단계에서 HTTP 자체를 이해하는 장이다. 요청/응답 규약을 직접 다뤄야 API 설계의 기준이 흔들리지 않는다.

## 왜 중요한가
Express를 쓰더라도 결국 HTTP 규약 위에서 동작한다. 프로토콜 이해가 부족하면 상태코드, 헤더, 캐싱, 에러 응답이 일관되지 않다.

## 핵심 개념
### 1) HTTP 메시지 구조
요청 라인, 헤더, 바디, 상태코드는 API 의미를 전달하는 기본 단위다. 이 구조를 이해해야 계약 문서와 구현이 일치한다.

### 2) 요청 처리 파이프라인
요청 수신부터 응답 반환까지 단계를 분리하면 병목 지점과 예외 지점을 명확히 볼 수 있다.

### 3) 응답 표준화
POST 데이터 처리와 검증에서 핵심은 기능 구현보다 응답 포맷, 상태코드, 에러 규칙을 팀 표준으로 만드는 것이다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트가 기대하는 응답 형태를 서버에서 먼저 고정할 수 있다.
- UI 에러 처리와 서버 상태코드를 정확히 매핑할 수 있다.
- 네트워크 디버깅 시간을 줄일 수 있다.

## 실무에서 자주 생기는 오해
- 상태코드 의미를 무시하고 모든 응답을 동일하게 반환한다.
- 입력 검증을 생략한 채 본문을 신뢰한다.
- 예외 처리 규칙 없이 핸들러마다 다른 응답을 만든다.

## 정리
POST 데이터 처리와 검증는 백엔드 API 품질의 기반이 되는 HTTP 이해 단원이다.
