# 테스트 해보기 단위 통합 부하 강의 03. Mock/Stub/Test Double 활용

## 학습 개요
Mock/Stub/Test Double 활용를 중심으로 테스트 피라미드 관점에서 품질 확보 전략을 정리한다.

## 핵심 개념
### 1) 테스트 층위 이해
단위/통합/부하 테스트는 서로 대체가 아니라 보완 관계다. 각 층의 목적을 분리해야 비용 대비 효과가 높다.

### 2) 회귀 방지 시스템
테스트는 기능 검증을 넘어 변경 안전장치다. 실패 재현 가능성을 코드로 남기는 것이 핵심이다.

### 3) 운영 전 검증
Mock/Stub/Test Double 활용에서는 성능 한계와 장애 조건을 배포 전에 확인해 실제 사고 확률을 낮춘다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트 변경이 백엔드 계약을 깨는지 자동으로 확인할 수 있다.
- 테스트를 기준으로 API 문서 신뢰도를 높일 수 있다.
- 배포 전 성능 리스크를 사전에 탐지할 수 있다.

## 실무에서 자주 생기는 오해
- E2E만 테스트하고 단위 테스트를 생략한다.
- 외부 의존성 경계를 분리하지 않아 flaky 테스트를 만든다.
- 부하 테스트 없이 운영 트래픽을 받는다.

## 정리
Mock/Stub/Test Double 활용는 품질을 감으로 관리하지 않고 측정 가능한 지표로 관리하게 만드는 단원이다.
