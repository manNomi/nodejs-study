# 타입스크립트로 전환해보기 강의 03. Express 타입 적용

## 학습 개요
Express 타입 적용는 Node 백엔드 코드에 타입 안정성을 도입하는 전략을 다룬다.

## 왜 중요한가
기능이 커질수록 런타임 오류를 사전에 차단할 장치가 필요하다. 타입 시스템은 변경 안정성을 높이는 핵심 도구다.

## 핵심 개념
### 1) 점진적 전환 전략
한 번에 전환하기보다 경계 모듈부터 타입을 도입해야 리스크를 줄일 수 있다.

### 2) 타입 계약 설계
API 입출력, 서비스 시그니처, 도메인 모델을 타입으로 명시하면 협업 비용이 줄어든다.

### 3) 런타임 검증과의 분리
Express 타입 적용에서 중요한 점은 타입 체크와 런타임 검증을 역할 분리해 함께 설계하는 것이다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트와 백 타입 계약을 공유해 변경 충돌을 줄인다.
- 컴파일 단계에서 영향 범위를 조기에 파악할 수 있다.
- 리팩터링 속도와 안정성을 동시에 확보할 수 있다.

## 실무에서 자주 생기는 오해
- any를 남발한다.
- 타입만 믿고 런타임 검증을 생략한다.
- 우선순위 없이 빅뱅 전환을 시도한다.

## 정리
Express 타입 적용는 백엔드 코드베이스를 장기 유지보수 가능한 구조로 바꾸는 핵심 단원이다.
