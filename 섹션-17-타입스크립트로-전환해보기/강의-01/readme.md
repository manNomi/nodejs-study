# 타입스크립트로 전환해보기 강의 01. 타입스크립트 전환 전략

## 학습 개요
타입스크립트 전환 전략를 통해 Node 백엔드 코드의 타입 안정성과 유지보수성을 높이는 기준을 정리한다.

## 핵심 개념
### 1) 점진적 전환 전략
기존 JavaScript 코드를 한 번에 바꾸기보다 경계 모듈부터 점진 전환하는 방식이 리스크를 줄인다.

### 2) 타입 시스템의 실무 가치
타입은 문법 장식이 아니라 계약 검증 도구다. API 입력/출력, 서비스 시그니처 안정성에 직접 기여한다.

### 3) 런타임 검증과의 역할 분리
타입스크립트 전환 전략에서 중요한 점은 컴파일 타입 체크와 런타임 검증을 함께 설계하는 것이다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트 타입과 백엔드 타입 계약을 공유해 변경 비용을 줄일 수 있다.
- API 스키마 변경 시 컴파일 단계에서 영향 범위를 파악할 수 있다.
- 리팩터링 속도와 안정성을 동시에 높일 수 있다.

## 실무에서 자주 생기는 오해
- any를 남발해 타입 전환 효과를 없앤다.
- 타입만 믿고 런타임 검증을 생략한다.
- 마이그레이션 우선순위 없이 전면 교체를 시도한다.

## 정리
타입스크립트 전환 전략는 백엔드 코드베이스를 장기 유지보수 가능한 구조로 전환하는 핵심 단원이다.
