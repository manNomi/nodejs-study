# 노드버드 SNS 만들기 강의 03. 게시글/업로드 도메인

## 학습 개요
게시글/업로드 도메인는 SNS 도메인을 통합적으로 설계하는 관점을 다룬다. 기능 단위 구현을 서비스 단위 품질로 연결하는 것이 핵심이다.

## 왜 중요한가
회원, 게시글, 팔로우, 피드, 댓글은 각각 독립 기능처럼 보이지만 실제로는 강하게 결합된 도메인 규칙이다.

## 핵심 개념
### 1) 도메인 경계 설계
기능 중심이 아니라 도메인 규칙 중심으로 경계를 정의해야 확장 시 혼란이 줄어든다.

### 2) 상호작용 흐름
게시글/업로드 도메인에서는 읽기/쓰기 흐름, 권한, 알림, 부수효과를 하나의 시나리오로 연결해 봐야 한다.

### 3) 운영 품질 개선
피드 성능, 업로드 안정성, 예외 처리 표준화는 기능 완료 이후 반드시 수행해야 할 단계다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- UI 요구사항을 도메인 제약과 함께 해석할 수 있다.
- 상태관리 충돌을 서버 규칙으로 조정할 수 있다.
- 기능 개발 이후 품질 개선 루틴을 만들 수 있다.

## 실무에서 자주 생기는 오해
- 기능별 구현만 하고 도메인 규칙 문서화를 생략한다.
- 피드 조회 성능을 후순위로 둔다.
- 부수효과 실패 시 보상 전략이 없다.

## 정리
게시글/업로드 도메인는 백엔드 기능을 서비스 운영 수준으로 끌어올리는 통합 단원이다.
