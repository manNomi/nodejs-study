# 노드버드 SNS 만들기 강의 04. 팔로우/언팔로우 구현

## 학습 개요
팔로우/언팔로우 구현를 통해 실서비스형 SNS 도메인을 통합적으로 설계하는 관점을 정리한다.

## 핵심 개념
### 1) 도메인 중심 기능 연결
인증, 포스트, 팔로우, 피드, 댓글은 독립 기능이 아니라 서로 제약을 주고받는 도메인 규칙 집합이다.

### 2) 기능에서 운영으로의 전환
기능 구현 완료 이후 성능, 보안, 일관성, 예외 처리 관점으로 재점검해야 실제 서비스가 된다.

### 3) 조회 최적화 전략
팔로우/언팔로우 구현에서는 피드/타임라인 조회 최적화가 핵심이다. 데이터 모델과 쿼리 전략을 함께 설계해야 한다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- UI 요구사항을 도메인 제약과 함께 해석할 수 있다.
- 프론트 상태관리와 서버 규칙 간 충돌을 줄일 수 있다.
- 기능 우선 개발 후 운영 품질 개선 루틴을 만들 수 있다.

## 실무에서 자주 생기는 오해
- 기능 단위 구현만 하고 도메인 규칙을 문서화하지 않는다.
- 조회 성능을 후순위로 미뤄 병목을 키운다.
- 부수효과(업로드/삭제/알림) 실패 시 보상 전략이 없다.

## 정리
팔로우/언팔로우 구현는 백엔드 기능을 서비스 수준으로 끌어올리는 통합 설계 단원이다.
