# CLI 프로그램 만들기 강의 01. CLI 명령 설계와 UX

## 학습 개요
CLI 명령 설계와 UX를 통해 CLI 도구 설계와 자동화 관점을 정리한다.

## 핵심 개념
### 1) 명령형 인터페이스 설계
CLI는 개발자 UX 도구다. 인자 설계, 도움말, 오류 메시지의 품질이 사용성을 좌우한다.

### 2) 자동화 파이프라인 연결
스크립트화 가능한 CLI는 반복 작업을 제거하고 배포/운영 자동화의 기반이 된다.

### 3) 표준 출력/종료코드
CLI 명령 설계와 UX에서 종료코드와 출력 채널(stdout/stderr)을 분리하면 CI 환경에서 신뢰성 있는 실행 결과를 얻을 수 있다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트 빌드/검증 작업을 CLI로 재사용 가능하게 만들 수 있다.
- 로컬 작업과 CI 작업을 동일 명령으로 통일할 수 있다.
- 개발자 도구 품질을 제품 품질과 연결할 수 있다.

## 실무에서 자주 생기는 오해
- 도움말 없이 내부용 스크립트처럼 만든다.
- 실패 시 종료코드 규칙이 없다.
- stderr/stdout을 섞어 자동화 파서가 깨진다.

## 정리
CLI 명령 설계와 UX는 개발 생산성을 시스템적으로 끌어올리는 자동화 단원이다.
