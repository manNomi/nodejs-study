# CLI 프로그램 만들기 강의 01. CLI 명령 UX 설계

## 학습 개요
CLI 명령 UX 설계는 개발자 도구로서 CLI를 설계하고 자동화 체계에 연결하는 기준을 다룬다.

## 왜 중요한가
반복 작업을 수동으로 처리하면 품질 편차와 시간이 동시에 증가한다. CLI는 이를 구조적으로 줄이는 수단이다.

## 핵심 개념
### 1) 명령 인터페이스 설계
인자 체계, 도움말, 오류 메시지는 CLI 사용성을 결정한다.

### 2) 자동화 연계
CLI 명령 UX 설계를 제대로 설계하면 로컬 작업과 CI 작업을 같은 명령 체계로 묶을 수 있다.

### 3) 종료코드/출력 규약
stdout/stderr 분리와 종료코드 정의는 자동화 환경 신뢰성의 핵심이다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트 빌드/검증 업무를 도구화할 수 있다.
- 개발 절차를 명령어 수준에서 표준화할 수 있다.
- 팀 온보딩 시간을 단축할 수 있다.

## 실무에서 자주 생기는 오해
- 도움말 없이 내부 스크립트처럼 제공한다.
- 실패 시 종료코드를 일관되게 주지 않는다.
- 출력 채널을 구분하지 않는다.

## 정리
CLI 명령 UX 설계는 개발 생산성을 시스템적으로 높이는 자동화 단원이다.
