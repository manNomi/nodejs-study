# 실시간 경매 시스템 만들기 서버센트이벤트 스케줄링 강의 01. SSE 연결 설계

## 학습 개요
SSE 연결 설계는 시간 기반 도메인에서 SSE와 스케줄링을 결합하는 설계를 설명한다.

## 왜 중요한가
경매/예약/마감 처리처럼 시간이 비즈니스 규칙인 시스템은 일반 CRUD보다 실패 시 영향이 크다.

## 핵심 개념
### 1) 서버 푸시 모델
SSE는 단방향 실시간 전송에 적합하다. 연결 유지 비용과 전송 전략을 함께 고려해야 한다.

### 2) 시간 기준 통일
서버 시간, 타임존, 마감 시점을 명확히 정의해야 비즈니스 충돌을 줄일 수 있다.

### 3) 스케줄링 신뢰성
SSE 연결 설계에서는 누락/중복 실행에 대비한 재처리 전략이 필수다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 클라이언트 타이머와 서버 확정 시점을 분리해 해석할 수 있다.
- 실시간 갱신 화면에서 확정 이벤트를 안정적으로 처리할 수 있다.
- 시간 관련 버그를 설계 단계에서 줄일 수 있다.

## 실무에서 자주 생기는 오해
- 클라이언트 시간 기준으로 로직을 확정한다.
- 실패 재처리 정책이 없다.
- SSE 연결 비용 계산을 생략한다.

## 정리
SSE 연결 설계는 시간·이벤트 결합 도메인을 안정적으로 운영하기 위한 단원이다.
