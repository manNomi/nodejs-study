# 실시간 경매 시스템 만들기 서버센트이벤트 스케줄링 강의 02. 경매 도메인 로직

## 학습 개요
경매 도메인 로직를 중심으로 시간 기반 도메인(SSE/스케줄링) 처리 방식을 정리한다.

## 핵심 개념
### 1) 서버 푸시 모델
SSE는 단방향 실시간 전송에 적합하다. 연결 유지 비용과 이벤트 전달 보장 전략을 함께 고려해야 한다.

### 2) 시간 조건 처리
경매/예약 같은 도메인은 마감 시점이 핵심 규칙이다. 서버 시간 기준과 재처리 전략이 정확해야 한다.

### 3) 스케줄링 신뢰성
경매 도메인 로직에서는 작업 누락, 중복 실행, 장애 복구를 전제로 스케줄러를 설계해야 한다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트 타이머와 서버 기준 시간을 분리해 해석할 수 있다.
- 실시간 갱신 화면에서 데이터 확정 시점을 명확히 다룰 수 있다.
- 시간 기반 버그(타임존/지연)를 설계 단계에서 줄일 수 있다.

## 실무에서 자주 생기는 오해
- 클라이언트 시간을 기준으로 비즈니스 결정을 내린다.
- 스케줄 실패 재처리 정책이 없다.
- SSE 연결 수 증가에 따른 자원 계산을 하지 않는다.

## 정리
경매 도메인 로직는 시간과 이벤트가 결합된 도메인을 안정적으로 운영하기 위한 단원이다.
