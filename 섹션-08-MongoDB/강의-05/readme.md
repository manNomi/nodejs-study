# MongoDB 강의 05. 집계 파이프라인과 인덱스

## 학습 개요
집계 파이프라인과 인덱스를 통해 문서지향 데이터 모델과 MongoDB 운영 기준을 정리한다.

## 핵심 개념
### 1) 문서지향 모델 사고방식
MongoDB는 관계보다 문서 단위 응집을 우선한다. 조회 패턴을 먼저 정의한 뒤 스키마를 설계해야 효율적이다.

### 2) 스키마와 검증
유연한 저장소일수록 검증 규칙이 더 중요하다. 스키마 제약이 약하면 운영 데이터 품질이 급격히 떨어진다.

### 3) 집계와 인덱스
집계 파이프라인과 인덱스에서는 단순 CRUD를 넘어 집계 파이프라인과 인덱스 설계까지 이해해야 실무 성능을 낼 수 있다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트에서 필요한 응답 형태를 문서 구조로 직접 반영할 수 있다.
- 관계형과 문서형의 선택 기준을 명확히 세울 수 있다.
- 빠른 기능 실험과 데이터 품질 사이 균형을 잡을 수 있다.

## 실무에서 자주 생기는 오해
- 문서 구조를 무제한 중첩한다.
- 검증 없이 유연성만 추구한다.
- 인덱스 없는 정렬/검색을 운영에 반영한다.

## 정리
집계 파이프라인과 인덱스는 유연성과 운영 안정성의 균형을 배우는 문서지향 데이터 단원이다.
