# MongoDB 강의 03. Mongoose 스키마 설계

## 학습 개요
Mongoose 스키마 설계는 문서지향 데이터 모델의 장단점과 MongoDB 운용 기준을 정리한다.

## 왜 중요한가
유연한 스키마는 빠른 개발에 유리하지만, 검증과 인덱스 전략이 없으면 데이터 품질과 성능이 빠르게 무너진다.

## 핵심 개념
### 1) 문서 중심 모델링
조회 패턴을 먼저 정의한 뒤 문서 구조를 설계해야 MongoDB의 장점을 살릴 수 있다.

### 2) 스키마와 검증
유연성은 무제한 자유가 아니다. Mongoose 스키마 설계에서는 스키마 검증 규칙이 운영 품질을 지킨다.

### 3) 집계와 인덱스 전략
단순 CRUD를 넘어 집계 파이프라인과 인덱스를 함께 설계해야 실무 성능을 얻는다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트 응답 요구 형태를 문서 구조에 직접 반영할 수 있다.
- 관계형과 문서형 선택 기준을 명확히 세울 수 있다.
- 빠른 기능 실험과 데이터 품질 사이 균형을 잡을 수 있다.

## 실무에서 자주 생기는 오해
- 문서를 과도하게 중첩한다.
- 검증 규칙 없이 저장 구조를 확장한다.
- 인덱스 없이 검색/정렬을 운영에 반영한다.

## 정리
Mongoose 스키마 설계는 문서지향 데이터 모델을 실무 수준으로 다루기 위한 핵심 단원이다.
