# API 서버 만들기 JWT CORS 강의 06. 통합 테스트/배포 준비

## 학습 개요
통합 테스트/배포 준비는 API 서버의 인증/보안/에러 계약을 표준화하는 데 초점을 둔다.

## 왜 중요한가
API는 프론트와 백엔드를 연결하는 계약이다. 인증과 에러 규칙이 흔들리면 화면 로직과 운영 로직이 동시에 불안정해진다.

## 핵심 개념
### 1) 인증/인가 구조
인증은 사용자 확인, 인가는 권한 판단이다. 두 레이어를 분리해야 보안 구조가 명확해진다.

### 2) 토큰 기반 운영
JWT는 발급만으로 끝나지 않는다. 만료, 재발급, 폐기, 클레임 설계를 함께 고려해야 한다.

### 3) 계약 표준화
통합 테스트/배포 준비에서는 CORS 정책, 상태코드, 에러 포맷을 팀 공통 규약으로 고정하는 것이 핵심이다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트 인증 상태와 서버 토큰 정책을 일치시킬 수 있다.
- 에러 응답 표준화로 UI 예외 처리 비용을 줄일 수 있다.
- 보안 정책을 개발 초기부터 반영할 수 있다.

## 실무에서 자주 생기는 오해
- 토큰 만료/재발급 전략 없이 인증을 구성한다.
- CORS를 무조건 전체 허용한다.
- HTTP 코드와 비즈니스 코드 경계를 섞는다.

## 정리
통합 테스트/배포 준비는 API 서버를 운영 가능한 보안/계약 체계로 만드는 핵심 단원이다.
