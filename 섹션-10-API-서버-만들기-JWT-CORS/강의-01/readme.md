# API 서버 만들기 JWT CORS 강의 01. API 서버 설계 원칙

## 학습 개요
API 서버 설계 원칙를 중심으로 API 서버의 인증/보안/응답 표준을 정리한다.

## 핵심 개념
### 1) 인증과 인가의 분리
인증은 '누구인지' 확인, 인가는 '무엇을 할 수 있는지' 판단이다. 두 경계를 분리해야 보안 구조가 명확해진다.

### 2) 토큰 기반 아키텍처
JWT를 사용할 때는 발급/검증/만료/재발급 전략을 함께 설계해야 운영에서 안정적으로 동작한다.

### 3) API 계약 표준화
API 서버 설계 원칙의 실무 핵심은 CORS, 에러 포맷, 상태코드를 팀 공통 규약으로 고정하는 것이다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트 인증 상태관리와 토큰 정책을 일관되게 연결할 수 있다.
- 에러 응답 스키마 표준화로 UI 예외 처리 비용을 줄인다.
- 보안 정책을 개발 단계에서부터 테스트할 수 있다.

## 실무에서 자주 생기는 오해
- 토큰 만료/재발급 전략 없이 인증을 도입한다.
- CORS를 무조건 전체 허용으로 설정한다.
- HTTP 상태코드와 비즈니스 에러코드를 혼용한다.

## 정리
API 서버 설계 원칙는 API 서버를 팀 단위로 운영 가능한 보안/계약 체계로 정리하는 단원이다.
