# AWS 서버리스 S3 Lambda 사용하기 강의 01. S3 업로드와 접근권한 설계

## 학습 개요
S3 업로드와 접근권한 설계를 중심으로 서버리스 아키텍처(S3+Lambda) 운영 기준을 정리한다.

## 핵심 개념
### 1) 이벤트 기반 실행 모델
서버리스는 요청 상시 대기가 아니라 이벤트 발생 시 함수가 실행되는 모델이다.

### 2) 권한과 보안
IAM 최소권한 원칙이 설계 중심이 된다. 권한 과다 부여는 즉시 보안 리스크로 이어진다.

### 3) 운영 제약 이해
S3 업로드와 접근권한 설계에서는 콜드스타트, 타임아웃, 재시도/중복 실행 같은 런타임 특성을 고려해야 한다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트 업로드 흐름과 백엔드 후처리를 분리 설계할 수 있다.
- 비용/확장성 요구사항에 맞춰 서버리스 채택 여부를 판단할 수 있다.
- 이벤트 기반 UI 업데이트 전략과 연결할 수 있다.

## 실무에서 자주 생기는 오해
- 함수 권한을 과도하게 부여한다.
- 재시도/중복 처리 전략 없이 이벤트를 처리한다.
- 타임아웃과 콜드스타트를 운영 설계에서 무시한다.

## 정리
S3 업로드와 접근권한 설계는 인프라 운영 방식을 이벤트 중심으로 전환하는 단원이다.
