# AWS 서버리스 S3 Lambda 사용하기 강의 01. S3 업로드/권한

## 학습 개요
S3 업로드/권한는 서버리스 아키텍처에서 이벤트 기반 처리 모델을 이해하는 장이다.

## 왜 중요한가
서버 상시 운영 모델과 달리 서버리스는 이벤트 트리거 중심으로 동작한다. 권한, 비용, 지연 특성을 함께 고려해야 한다.

## 핵심 개념
### 1) 이벤트 기반 실행
S3 업로드 같은 이벤트가 Lambda 실행을 촉발한다. 흐름을 이벤트 체인으로 이해해야 설계가 단순해진다.

### 2) 권한과 보안
IAM 최소 권한 원칙은 서버리스에서 필수다. 권한 과다 부여는 즉시 보안 리스크가 된다.

### 3) 운영 제약
S3 업로드/권한에서는 콜드스타트, 타임아웃, 재시도/중복 실행을 전제로 설계해야 한다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트 업로드 흐름과 백엔드 후처리를 분리할 수 있다.
- 비용/확장성 관점에서 서버리스 채택 기준을 세울 수 있다.
- 이벤트 기반 UI 갱신과 연결 지점을 정의할 수 있다.

## 실무에서 자주 생기는 오해
- 권한을 과도하게 부여한다.
- 중복 실행 대비 로직이 없다.
- 타임아웃/콜드스타트를 고려하지 않는다.

## 정리
S3 업로드/권한는 인프라 운영 방식을 이벤트 중심으로 전환하는 핵심 단원이다.
