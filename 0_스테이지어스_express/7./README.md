# 7. 18주차 정리

## 주차 목표

- 프로세스 운영 관점에서 백엔드 실행 전략 이해
- AWS 핵심 서비스(EC2/EBS/S3/RDS) 용도 구분
- 파일 업로드 API 설계(Multer)까지 연결

## 핵심 개념

### 1) Process와 운영

- 프로세스는 OS에서 실행 중인 프로그램 단위
- 멀티 프로세스는 안정성/성능 측면에서 서버 운영에 중요
- `pm2`를 활용하면 백그라운드 실행, 재시작, 모니터링 자동화 가능

주요 명령 예시:

- `pm2 start server.js`
- `pm2 restart server.js`
- `pm2 delete server.js`

### 2) AWS 기본 서비스 구분

- EC2: 서버 인스턴스(컴퓨팅)
- EBS: 인스턴스 부착형 블록 스토리지
- S3: 객체 스토리지(대용량 파일, 정적 리소스)
- RDS: 관리형 데이터베이스 서비스

### 3) 스토리지 선택 기준

- 코드/애플리케이션 실행 자원: EC2 + EBS
- 사용자 업로드 파일(이미지/영상): S3
- 관계형 데이터 운영: RDS

### 4) 운영 실무 메모

- EBS 확장 시 파일시스템 리사이즈 작업까지 필요
- S3는 버킷/객체 단위로 관리하며 URL 접근 모델 이해 필요
- IAM 권한 정책을 최소 권한 원칙으로 설정

## 과제 요구사항 정리

Multer 기반 첨부파일 API 구현:

- 파일 1개 업로드/다운로드 API
- 텍스트 + 이미지 1개 처리 API
- 텍스트 + 이미지 여러 개 처리 API
- 텍스트/이미지 수정 API

## 코드 파일 정리

이 주차는 실습 코드가 함께 포함되어 있으며 아래 경로에 보존됨:

- `/18주차/index.html`
- `/18주차/router/pages.js`
- `/18주차/pages/*.ejs`
- `/18주차/pages/html/*.html`
- `/18주차/pages/err_page/404.html`

## 실무 관점 체크포인트

- 업로드 기능은 저장소 전략(S3 연계), 파일 검증, 접근권한까지 함께 설계해야 함
- 단일 서버 운영에서 시작하더라도 프로세스 관리 습관(pm2, 로그, 재시작 정책)을 초기에 잡는 것이 중요
