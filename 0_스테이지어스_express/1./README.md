# 1. 12주차 정리

## 주차 목표

- 심화 단계 시작: 단순 구현이 아니라 현업 기준의 구조/패턴/예외처리까지 고려
- Node.js + Express 기반 백엔드 서버 뼈대 수립
- 프론트 요청을 처리할 API 체계와 REST 설계 규칙 학습

## 핵심 개념

### 1) Node.js와 Express의 역할

- Node.js는 백엔드 전용 언어가 아니라 JavaScript Runtime
- 브라우저 외 환경에서도 JavaScript를 실행할 수 있게 만들어 준 플랫폼
- Express는 Node.js 위에서 HTTP 서버와 라우팅을 빠르게 구성하는 웹 프레임워크

### 2) ES6+와 백엔드 JavaScript

- 백엔드에서는 JavaScript 사용 비중이 사실상 100%
- ES6+ 문법은 단순 문법 편의가 아니라, 스코프/재선언/호이스팅 문제 개선과 유지보수성 향상에 직결
- 수업 외에도 문서 기반으로 문법/성능/내부 동작을 병행 학습해야 함

### 3) API와 REST 설계

- API는 내부 코드를 공유하는 방식이 아니라, 외부가 호출할 수 있는 인터페이스 제공 방식
- REST는 API 명명/행위 설계 규칙이며, HTTP Method를 동작 의미로 활용
- 핵심 메서드:
  - `GET`: 조회
  - `POST`: 생성
  - `PUT`: 수정
  - `DELETE`: 삭제

### 4) Express 프로젝트 기본 구조

- `package.json`: 의존성과 실행 정보 명세
- `node_modules`: 설치된 패키지 저장
- `package-lock.json`: 설치 시점 버전 고정
- `router/`: API 엔드포인트 분할 관리
- `server.js`: 서버 실행 진입점

## 과제 요구사항 정리

8주 기반 프로젝트 시작점으로 아래 기능을 API 형태로 준비:

- 인증/계정: 로그인, 회원가입, 아이디 찾기, 비밀번호 찾기
- 회원 기능: 프로필 조회, 회원정보 수정, 회원탈퇴
- 게시판: 게시글 CRUD
- 댓글: 댓글 CRUD
- API 테스트: Postman으로 검증

## 보강 메모

- `express()`는 메인 앱 인스턴스 생성
- `express.Router()`는 기능 단위 미니 라우터 분리
- `cookieParser()`는 쿠키 파싱 미들웨어
- `writeHead(200, {...})`는 정상 응답 헤더 전송(HTTP 상태코드 포함)

## 실무 관점 체크포인트

- 과제 구현보다 중요한 것은 "왜 이 구조가 유지보수에 유리한가"
- 백엔드는 예외처리가 최종 방어선이라는 전제로 설계 시작
- 이후 주차에서 인증/DB/캐시/배포로 확장 가능한 API 네이밍을 초기에 통일
