# 05. 노드의 특성

## 학습 개요
노드의 특성를 통해 Node.js가 어떤 실행 환경인지, 그리고 브라우저 JavaScript와 무엇이 다른지를 백엔드 관점에서 정리한다.

## 핵심 개념
### 1) 런타임 관점에서의 Node.js
Node.js는 프레임워크가 아니라 JavaScript 런타임이다. 이 관점을 먼저 잡아야 이후의 모듈, 서버, 배포 개념이 정확히 연결된다.

### 2) 이벤트 루프 기반 처리 모델
Node.js는 이벤트 루프와 비동기 I/O를 중심으로 동작한다. 따라서 요청이 많은 서비스에서 효율을 얻을 수 있고, CPU 중심 작업에서는 별도 전략이 필요하다.

### 3) 프론트-백 역할 분리
같은 JavaScript라도 브라우저 API와 서버 API는 책임이 다르다. 노드의 특성를 이해하면 클라이언트 문제와 서버 문제를 구분하는 기준이 생긴다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- UI 지연의 원인이 렌더링인지 서버 처리인지 분리해서 분석할 수 있다.
- API를 소비하는 입장에서 생산하는 입장으로 사고를 확장할 수 있다.
- 인증, 에러 포맷, 캐싱 같은 경계 이슈를 서버 쪽에서 직접 제어할 수 있다.

## 실무에서 자주 생기는 오해
- Node.js를 프레임워크라고 오해해 학습 순서를 잘못 잡는다.
- 싱글 스레드라는 말만 보고 성능 특성을 단순화한다.
- 실행 환경 차이(브라우저/서버)를 구분하지 않고 코드 책임을 섞는다.

## 정리
노드의 특성는 Node.js 학습의 출발점이며, 이후 모든 백엔드 개념의 좌표계를 맞추는 기본 장이다.
