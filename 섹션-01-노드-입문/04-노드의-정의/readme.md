# 04. 노드의 정의

## 학습 개요
노드의 정의는 Node.js를 서버 런타임으로 이해하기 위한 출발점이다. 이 섹션은 '브라우저 자바스크립트'와 '서버 자바스크립트'가 왜 같은 언어이면서도 다른 책임을 가지는지 설명한다.

## 왜 중요한가
이 단계를 제대로 이해하면 이후 Express, DB, 배포를 배울 때 기술을 단편적으로 외우지 않고 실행 모델 관점에서 연결할 수 있다.

## 핵심 개념
### 1) 런타임과 프레임워크의 구분
Node.js는 프레임워크가 아니라 런타임이다. 즉 애플리케이션 구조를 강제하지 않고, 자바스크립트가 서버에서 실행될 수 있는 환경과 API를 제공한다.

### 2) 이벤트 루프 기반 실행
요청을 처리할 때 핵심은 이벤트 루프와 비동기 I/O다. Node.js는 대기 시간을 활용해 여러 연결을 동시에 처리하는 구조를 갖는다.

### 3) 서버 관점의 자바스크립트
브라우저에서는 DOM이 중심이지만 서버에서는 네트워크, 파일, 프로세스가 중심이다. 노드의 정의를 이해하는 과정은 이 관점 전환을 만드는 과정이다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트에서 느린 API를 만났을 때, 렌더링 문제가 아니라 서버 처리 모델 문제인지 구분하는 기준이 생긴다.
- 클라이언트 상태와 서버 상태의 책임 경계를 나눠 설계할 수 있다.
- 백엔드 팀과 논의할 때 '동작 여부'를 넘어서 실행 구조 관점으로 대화할 수 있다.

## 실무에서 자주 생기는 오해
- Node.js를 Express와 동일한 개념으로 오해한다.
- 싱글 스레드라는 말만 보고 처리 성능을 단정한다.
- 브라우저 API와 서버 API의 경계를 섞어 이해한다.

## 정리
노드의 정의는 Node.js의 정체를 잡아주는 기초 단원이며, 이후 백엔드 학습의 공통 좌표계를 만든다.
