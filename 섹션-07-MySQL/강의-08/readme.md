# MySQL 강의 08. 인덱스와 성능

## 학습 개요
인덱스와 성능는 관계형 데이터 모델과 SQL 기반 서버 설계를 다룬다. 데이터 정합성과 조회 성능을 함께 확보하는 기준을 정리한다.

## 왜 중요한가
백엔드 장애의 많은 비율이 데이터 계층에서 발생한다. 모델링, 트랜잭션, 인덱스 이해가 없으면 기능이 늘수록 문제도 커진다.

## 핵심 개념
### 1) 관계형 모델링
테이블 구조, 키, 제약조건은 비즈니스 규칙을 데이터에 반영하는 장치다. 설계가 곧 품질이다.

### 2) 정합성 관리
트랜잭션 경계를 명확히 해야 동시 요청에서 데이터 불일치를 막을 수 있다.

### 3) 쿼리 성능 최적화
인덱스와 성능의 핵심은 CRUD 구현보다 JOIN/인덱스/실행 계획을 통한 병목 제거다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트의 리스트/검색/정렬 요구를 SQL 관점에서 최적화할 수 있다.
- 데이터 제약과 사용자 에러 메시지를 연결할 수 있다.
- 응답 지연 원인을 API 계층과 DB 계층으로 분리할 수 있다.

## 실무에서 자주 생기는 오해
- N+1 조회 문제를 방치한다.
- 트랜잭션 없이 다중 변경 작업을 수행한다.
- 인덱스 전략 없이 운영 트래픽을 받는다.

## 정리
인덱스와 성능는 데이터 정합성과 성능을 동시에 다루는 관계형 백엔드의 중심 단원이다.
