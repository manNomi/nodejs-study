# MySQL 강의 01. 관계형 데이터 모델링 기초

## 학습 개요
관계형 데이터 모델링 기초를 중심으로 관계형 데이터 모델과 SQL 기반 백엔드 설계를 정리한다.

## 핵심 개념
### 1) 데이터 모델링
테이블/키/관계를 어떻게 설계하느냐가 기능 확장성과 조회 성능을 결정한다.

### 2) 트랜잭션과 정합성
동시 요청 환경에서는 트랜잭션 경계를 명확히 해야 데이터 불일치를 막을 수 있다.

### 3) 쿼리 성능
관계형 데이터 모델링 기초 학습의 핵심은 CRUD 자체보다 JOIN, 인덱스, 실행 계획을 통한 병목 제거다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 프론트 pagination/filter 요구사항을 SQL 레벨에서 최적화할 수 있다.
- 에러를 DB 제약조건과 연결해 사용자 메시지로 변환할 수 있다.
- 조회 API 성능 이슈를 네트워크 탓으로만 돌리지 않게 된다.

## 실무에서 자주 생기는 오해
- N+1 조회 문제를 방치한다.
- 트랜잭션 없이 다중 변경 쿼리를 실행한다.
- 인덱스 없이 운영 트래픽을 받는다.

## 정리
관계형 데이터 모델링 기초는 데이터 정합성과 성능을 동시에 확보하기 위한 관계형 백엔드의 핵심 단원이다.
