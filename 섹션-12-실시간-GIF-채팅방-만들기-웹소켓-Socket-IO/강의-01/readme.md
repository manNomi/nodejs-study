# 실시간 GIF 채팅방 만들기 웹소켓 Socket IO 강의 01. 웹소켓 연결과 생명주기

## 학습 개요
웹소켓 연결과 생명주기를 통해 실시간 통신 서버의 상태관리와 이벤트 설계 원칙을 정리한다.

## 핵심 개념
### 1) 연결 중심 사고
HTTP 요청-응답과 달리 WebSocket은 연결 상태 자체가 도메인 상태가 된다. 연결/재연결/해제 전략이 필수다.

### 2) 이벤트 계약
이벤트 이름, payload 스키마, ACK 규칙을 정의해야 프론트-백 실시간 기능이 안정적으로 동작한다.

### 3) 브로드캐스트 범위 제어
웹소켓 연결과 생명주기에서는 room/namespace 단위 설계가 트래픽과 보안을 동시에 좌우한다.

## 프론트엔드에서 백엔드로 확장할 때의 관점
- 실시간 UI 상태와 서버 이벤트를 정합성 있게 동기화할 수 있다.
- 지연/유실/재연결 케이스를 화면 설계 단계에서 고려할 수 있다.
- 폴링 기반 구조를 실시간 구조로 전환할 판단 기준이 생긴다.

## 실무에서 자주 생기는 오해
- 재연결 시 상태 복구 전략을 준비하지 않는다.
- 이벤트 payload 계약을 문서화하지 않는다.
- 전체 브로드캐스트로 불필요한 트래픽을 만든다.

## 정리
웹소켓 연결과 생명주기는 실시간 기능을 데모 수준에서 운영 수준으로 올리는 핵심 단원이다.
