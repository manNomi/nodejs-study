## 교과서 핵심 개념
### A. 문제 상황 정의
실무에서는 기능 구현 자체보다, 실패했을 때 원인을 빠르게 찾고 복구하는 구조가 중요합니다. 채팅 브로드캐스트 구조는 이 문제를 해결하기 위한 기본 축입니다.

### B. 동작 메커니즘
소켓 핸드셰이크, room, namespace, 이벤트 ACK 관점에서 보면, 채팅 브로드캐스트 구조는 입력 검증, 실행 분기, 상태 전이, 응답 표준화 과정에 직접 관여합니다. 즉 "동작"뿐 아니라 "운영 안정성"까지 결정합니다.

### C. 설계 기준
- 경계 분리: 라우팅/비즈니스/저장소 책임 분리
- 실패 우선: 정상 흐름보다 실패 흐름을 먼저 설계
- 관측 가능성: 로그/메트릭/트레이스에 남는 구조
